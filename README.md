**🔗 Живой SaaS:** https://signalos-es52.onrender.com · **Демо (Pages):** https://masteroch.github.io/signalos/ · **Репозиторий:** https://github.com/MASTEROCH/signalos

> _Примечание: технические имена (Python-пакет `signalos/`, репозиторий `MASTEROCH/signalos`, Render-сервис) пока сохраняют старое имя — переименование инфраструктуры сделаем отдельным шагом._


**Цель:** находить **реальных клиентов для любого бизнеса — ненавязчиво**, без бюджета на рекламу и без своего блога.
Ядро (PULL): радар слушает публичные площадки и ловит людей, которые *прямо сейчас* пишут о боли,
которую решает твой продукт; оценивает **резонанс** (живой запрос vs сухое упоминание), готовит человечный
ответ, а утром присылает дайджест «🛰 N искр готовы» в Telegram. Ты только одобряешь и отвечаешь.
Дальше наращиваем PUSH-модули: контент, посты, рассылка, аналитика.

## Почему это работает (и почему не спам)
Машина **слушает и пишет черновик** — человек **одобряет и отправляет** со своего аккаунта.
Никакого массового парсинга личных данных и авто-рассылки. Только публичные сообщения +
ручное одобрение → ты остаёшься живым человеком, а не спам-ботом. Бюджет = копейки за API
вместо рекламы. Часто и вовсе бесплатно.

## Запуск за 60 секунд (ноль зависимостей, ноль ключей)
```bash
cd ~/code/signalos
./run.sh                      # запустит сервер и откроет дашборд
```
Открывается http://localhost:8000 → жми **«Сканировать»** → радар проходит по источникам
и кладёт сигналы в очередь триажа. Всё. Ядро — чистый Python (нужен только `python3`).

### Почему Telegram выключен и как включить
У Telegram нет бесплатного публичного API для чтения чатов (в отличие от остальных). Нужен одноразовый вход своим аккаунтом:
1. https://my.telegram.org → API development tools → создай приложение → получишь `api_id` и `api_hash`
2. Впиши их в `.env`: `TELEGRAM_API_ID=...` и `TELEGRAM_API_HASH=...`
3. `pip install telethon` затем `python3 -m signalos.tg_login` (введёшь телефон + код один раз)
4. В `config/config.json` у `telegram` поставь `"enabled": true` и добавь `"chats": [...]`

Только чтение публичных чатов, без рассылки.

## Апгрейды (опционально)
- **Claude** — задай `ANTHROPIC_API_KEY` (в `.env`) → классификация точнее, черновики живее.
  Без ключа работает бесплатный режим (скоринг по намерению + шаблонный черновик).
- **Telegram** — для русскоязычной аудитории, лучший канал именно под продукты Роча.

## Настройка
`config/config.json` (создаётся из `config.example.json` при первом запуске):
- `projects[]` — под что ловим: ключевики, tone of voice, ссылка, аудитория. Мульти-проект.
- `sources[]` — где слушаем: вкл/выкл, сабреддиты, RSS-ленты, TG-чаты.

Это же ядро будущей **SaaS-версии**: чужой фаундер добавляет свой проект → платит немного.

## Дашборд
Очередь триажа в стиле Superhuman: `J`/`K` навигация, `A` одобрить+скопировать,
`E` править, `S` пропустить, `R` сканировать. «Сила сигнала» 1–5, причина сигнала,
ссылка на оригинал, язык, фильтр по проектам.
